# Demo Screenshots

Real screenshots captured via Playwright against a live `npm run start`
frontend server and a live `uvicorn` backend server, both running in the
same development sandbox this project was built in. None of these are
mockups or fabricated — each was generated by actually driving the real UI
(file upload via the real `<input type="file">`, real form submission, real
SSE streaming) and capturing the resulting page.

| File | What it shows |
|---|---|
| `01_empty_state.png` | The workbench before any document is uploaded — three-pane layout, empty states in all three panels. |
| `02_document_uploaded_pipeline.png` | The real ingestion pipeline in progress after uploading the sample PDF: Parsing → Structuring chunks → Embedding & indexing → Indexing keywords, each step's status updating live from real backend API responses. |
| `03_answer_streaming.png` | A question mid-stream: partial answer text with a live citation chip already rendered inline and the ink-caret streaming indicator. |
| `04_answer_with_exhibit_ledger.png` | The completed answer with the Exhibit Ledger panel populated and one citation tab expanded, showing the exact page/document snippet. |
| `05_evaluation_report_html.png` | The top portion of a real generated `evaluation/results/report.html`, including its inline SVG aggregate-metrics chart. |

## One honest caveat, stated once here and in `frontend/README.md`

The **retrieval pipeline** shown in these screenshots (real PDF parsing,
real chunking, real hybrid search, real MMR, real re-ranking, real
citation formatting) is entirely genuine production code. The **LLM answer
text** in screenshots 03 and 04 was produced by a throwaway demo harness
script (not part of this repository) that substitutes a canned-but-
content-aware response, because this development sandbox has no
`ANTHROPIC_API_KEY` configured — the network path to Anthropic is open
(confirmed reachable), only the credential is missing. Run the app with a
real key to see genuine Claude-generated answers in the same UI.
